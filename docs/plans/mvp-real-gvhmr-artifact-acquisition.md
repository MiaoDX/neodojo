# MVP Real GVHMR Artifact Acquisition Plan

Status: BLOCKED ON EXTERNAL GPU OPERATOR

## Goal

Produce the first non-fixture `neodojo.gvhmr_smplx_joints.v1` export for a
short local Baduanjin proof clip, import it with the existing real-conversion
lane, and make `make verify-real` pass without committing source media,
checkpoints, model assets, returned JSON, rendered videos, or generated outputs.

This is the operational completion plan for the remaining real-conversion gap.
The local command surface, GPU handoff package, validation, import-demo, audit,
self-hosted workflow intake, and guarded Pages promotion are already
implemented by the earlier plans; the missing artifact is the actual GPU-run
GVHMR result.

## Inputs

- A local/user-supplied source video path with rights reviewed by the operator.
- A materialized or package-archive handoff produced by one of:
  - `make real-gpu-operator-package-archive LOCAL_VIDEO=... REAL_LOCAL_SOURCE_ID=...`
  - `make real-gpu-colab-notebook LOCAL_VIDEO=... REAL_LOCAL_SOURCE_ID=...`
  - `make real-gpu-run-request LOCAL_VIDEO=... REAL_LOCAL_SOURCE_ID=...`
- Private GPU runtime with GVHMR dependencies, checkpoints, and licensed local
  SMPL-X model assets.
- The generated GPU-side `run_gvhmr_neodojo.sh` wrapper and
  `export_neodojo_gvhmr.py` helper from the operator package.

## Outputs

- Ignored returned `gvhmr-smplx-joints.json` with:
  - `schema: neodojo.gvhmr_smplx_joints.v1`
  - `fixture_only: false`
  - source provenance matching the materialized source clip
  - frame/timing data accepted by `real-conversion validate-source`
- Ignored `outputs/real-demo/` generated by `make real-artifact-intake`.
- Ignored `outputs/real-conversion-audit/manifest.json` with `complete: true`
  and `blocked: false`.
- Optional self-hosted `neodojo-self-hosted-real-demo` artifact or guarded Pages
  promotion artifact, only when explicitly enabled.

## Execution Tasks

1. Prepare the operator package from the selected local source:
   `make real-gpu-operator-package-archive LOCAL_VIDEO=... REAL_LOCAL_SOURCE_ID=...`.
2. Validate the ignored package archive locally:
   `make gvhmr-operator-package-archive-validate GVHMR_OPERATOR_PACKAGE_ARCHIVE=outputs/gvhmr-operator-package-archive`.
3. Move the package archive to a private GPU runtime or self-hosted GPU runner.
4. On the GPU runtime, install GVHMR dependencies, provide checkpoints and
   licensed SMPL-X assets, unpack the archive, run
   `run_gvhmr_neodojo.sh --install`, and export `gvhmr-smplx-joints.json`.
5. Confirm the returned JSON is GPU-generated, not a template or fixture smoke
   file, and contains `fixture_only: false`.
6. Return the JSON to the local workspace under ignored outputs.
7. Run:
   `make real-artifact-intake REAL_ARTIFACT_SOURCE_MATERIALIZATION=outputs/gvhmr-gpu-input/source-materialization.json REAL_ARTIFACT_GVHMR_JSON=path/to/gvhmr-smplx-joints.json`.
8. Run `make verify-real`.
9. If the strict audit passes and publication is desired, use the guarded
   real-demo Pages promotion workflow with explicit confirmation.

## Non-Goals

- Running GVHMR on the default local macOS CPU workspace.
- Running GVHMR on default GitHub-hosted CI.
- Committing source video, returned JSON, generated motion files, model assets,
  checkpoints, rendered media, logs, or other large outputs.
- Replacing the fixture-only public demo before a strict real audit passes.
- Treating a fixture smoke export as evidence of real conversion.

## Stop Condition

Stop when the first returned non-fixture GVHMR export is validated and imported,
`make verify-real` passes, and the real-demo evidence is documented without
tracking generated media or artifacts. Until then, this plan remains blocked on
an external GPU operator action, even though the local command surface and CI
fixture lane are complete.
